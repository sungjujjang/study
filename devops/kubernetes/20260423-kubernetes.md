# K8S
쿠버네티스의 준말, 여러 개의 노드를 사용해 컨테이너를 배포, 안정적이게 운영할 수 있도록 하는 관리 시스템  
**후술할 내용에서 팟 = 파드이다.**

## 클러스터
- 가장 큰 단위
- 가상 서버들이 속한 클라우드
- 여러 대의 노드로 구성됨

## 노드
- 각각 하나의 서버
- 쿠버네티스에서 컨테이너를 실행시키는 서버
- 마스터 노드와 워커 노드로 분리
  - 마스터 노드
    - 워커 노드들을 전체 관리
    - 1대만 띄워 놓았다가 죽으면 안 되므로 3대정도 띄운다고 함
  - 워커 노드
    - 배포하고자 하는 어플리케이션의 실제 실행을 수행한다

## 팟
- 가장 작은 배포 단위, 하나 이상의 컨테이너
- 동일한 네임스페이스 공유, 서로 간 통신 가능
- 동일한 아이피, 포트 공유
- 파드 내 어플리케이션은 같은 어플리케이션의 구성 요소로 묶여 있음

## 서비스
여러 파드를 통해 실행되는 애플리케이션을 네트워크에 노출시키는 가상 컴포넌트

## 네트워킹 방식
쿠버네티스는 파드 단위로 컨테이너들을 관리함  
- 아이피 할당 방식
  - 모든 팟은 고유한 IP를 가진다
  - 팟 내부의 컨테이너들은 이 아이피를 공유함
  1. Cluster CIDR - 클러스터 전체 아이피 대역 지정 (10.244.0.0/16)
  2. Node CIDR - 노드들은 위 Cluster CIDR에서 아이피를 할당받는다 (10.244.1.0/24)
  3. POD IP 할당 - 각각의 팟에 노드 아이피 대역에서 가져온 아이피를 할당

## 오토스케일링
HPA 오토스케일링은 파드의 개수를 조절해 부하를 분산하는 것

## 로드밸런싱
서비스의 부하를 분산시켜줌  
요청을 받을 때 로드밸런서가 받고, 요청을 받은 로드밸런서가 파드에 나누어 줌
- MetalLB
  - 온프레미스 환경에서 로드밸런서를 사용할 수 있도록 해주는 것
