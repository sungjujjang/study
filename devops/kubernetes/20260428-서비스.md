# 서비스
- 파드 간 네트워크의 연결을 담당하는 오브젝트
- 외부 네트워크에 파드들을 노출하는 오브젝트

## 왜 서비스를 쓰는가?
- 파드에 문제가 발생하면 쿠버네티스는 파드를 자동으로 삭제하고 다시 만듦
- 수동으로 새로 생긴 파드를 네트워크에 등록하기 번거로워 서비스를 사용
- 서비스를 사용하면 자동으로 네트워크 연결해줌

## 서비스 종류
- 클러스터 내부에서 파드 - 파드
  - ClusterIP
  - HeadLess
- 클러스터 외부에서 외부 - 내부
  - 노드포트
  - 로드밸런서

## Endpoint
- 서비스를 거쳐 실제로 도착하는 지점 (파드의 아이피)

## 로드밸런싱
- 요청(부하)를 분산하는 시스템
- 클러스터 아이피 서비스의 로드밸런싱
  - 내부 파드들의 통신 분산으로 자동적인 로드밸런싱
- 로드밸런서 타입의 서비스
  - 외부 접속을 분산
  1. GCP, AWS
    - 자동으로 로드밸런서를 연결해 줌
    - 로드밸런서로 서비스 선언만 하면 아이피를 받아옴
  2. On-Promise
    - 수동으로 MetalRB 등을 설치해야 함
    - 설치 후에는 GCP, AWS와 같이 동작함
