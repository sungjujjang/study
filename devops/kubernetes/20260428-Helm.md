# Helm
- 쿠버네티스의 차트를 관리하기 위한 도구
- 차트는 사전 구성된 쿠버네티스의 패키지
- 개발 환경마다 다른 정보를 관리하기에 유용
  - 데이터베이스 주소, api 키 등등

## 구조
- templates/
  - service.yaml
  - deployment.yaml
  - yaml 오브젝트 파일들이 들어가 있음
- Chart.yaml
  - 기본적인 메타 정보 정의
- values.yaml
  - 템플릿에 사용될 변수값들을 정의

## 패키지 관리
우분투의 apt처럼, 복잡한 쿠버네티스 리소스들을 설치/관리해줌

## 예시(로드밸런싱)
`value.yaml`  
```
service:
  type: LoadBalancer
  port: 80
```

`templates/service.yaml`  
```
apiVersion: v1
kind: Service
metadata:
  name: {{ include "my-web.fullname" . }}
spec:
  type: {{ .Values.service.type }} # values.yaml에서 값을 가져옴
  ports:
    - port: {{ .Values.service.port }}
  selector:
    app: {{ include "my-web.name" . }}
```

## .tpl(템플릿)이란
yaml 파일을 만들 때 반복적으로 쓰이는 문구를 모아놓은 곳
```
{{/* _helpers.tpl 파일 내용 예시 */}}
{{- define "my-web.fullname" -}}
{{- $name := default .Chart.Name .Values.nameOverride -}}
{{- if contains $name .Release.Name -}}
{{- .Release.Name | trunc 63 | trimSuffix "-" -}}
{{- else -}}
{{- printf "%s-%s" .Release.Name $name | trunc 63 | trimSuffix "-" -}}
{{- end -}}
{{- end -}}
```
- 위와 같이 정의한 후
- `name: {{ include "my-web.fullname" . }}` 이렇게 쓴다.
