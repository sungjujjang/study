# DHCP 프로토콜
동적으로 IP 주소를 할당해 주는 프로토콜  
일정 기간동안 특정 기기에서 IP를 사용하게 해 줌  
UDP 패킷을 이용, 비 연결형 서비스이다

## DHCP의 동작 방식
1. DHCP Discover
  - 클라이언트 > 서버
  - DHCP 서버를 찾기 위해 DHCP Discover 메시지를 **브로드캐스트**로 전송
2. DHCP Offer
  - 서버 > 클라이언트
  - 클라이언트에 할당할 IP주소와 서브넷, 게이트웨이 등을 클라이언트로 전송.
3. DHCP Request
  - 클라이언트 > 서버
  - 할당받은 아이피를 클라이언트에 등록, 아이피를 사용하겠다고 요청을 보냄.
4. DHCP Acknowledgement
  - 서버 > 클라이언트
  - 클라이언트 요청을 받으면 서버에 해당 아이피를 기록
  - DHCP 리퀘스트로 정상적으로 받았다고 등록
