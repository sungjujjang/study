# NOSQL
- not-only SQL
- SQL만을 사용하지는 않는 데이터베이스 시스템
- 하나의 유형이 아닌 SQL을 사용하지 않는 모든 데이터베이스 시스템이므로 종류가 많음

## 문서 지향 데이터베이스
JSON 객체와 비슷한 형태로 데이터를 저장
```
{
  "_id": "12345",
  "name": "foo bar",
  "email": "foo@bar.com",
  "address": {
    "street": "123 foo street",
    "city": "some city",
    "state": "some state",
    "zip": "123456"
  },
  "hobbies": ["music", "guitar", "reading"]
}
```
- MongoDB
- Couchbase

## 키-값 데이터베이스
캐싱이나 세션 관리에 주로 사용됨
```
Key: user:12345
Value: {"name": "foo bar", "email": "foo@bar.com", "designation": "software developer"}
```
- Redis
- Amazon Dynamo DB


## NoSql 장단점
- 장점
  - 스키마가 없기 때문에 유연하다. (비형식적인 데이터를 저장 가능하다)
  - 데이터를 읽는 속도가 빠르다.
- 단점
  - 데이터 중복이 존재한다
