# RDBMS (RDB, 관계형 데이터베이스)
모든 데이터를 2차원을 테이블 형태로 정의함
- 테이블
  - 열과 행으로 이루어진 표 형태
  - 최소 단위로 RDB는 여러 테이블로 이루어져 있음
- 열
  - 저장하고자 하는 데이터의 필드(카테고리)
  - 테이블은 각각의 속성을 가진 여러 열로 표현됨
- 행
  - 실질적인 진짜 1줄의 데이터
  - 행 데이터 또는 레코드(record)라고 부름
- 기본 키
  - 각 행을 구분하는 유일의 열
  - 테이블에서 중복될 수 없고, 비어 있을 수도 없음
  - 테이블마다 1개만 지정(없어도 됨)

## 외래 키(Foreign Key)
- **다른 테이블의 기본 키를 참조하는 키**

**외래 키 관계**
- 1:1
  - 한 사람이 하나의 주민번호를 가지는 경우
  - 보통 어느 쪽에 외래 키를 두어도 상관 없지만 주가 되는 테이블(위에서는 사람)에 둔다
- 1:N
  - 하나에 여러 레코드가 포함되는 경우
  - 하나의 부서에 여러 사람이 포함됨
  - 다(N) 쪽에 외래 키를 두어 일(1) 쪽을 참조함
- N:M
  - 학생 여러명이 강의 여러 개를 듣는 경우
  - 중간에 연결 테이블을 만들어 가져옴
  - 거의 쓰이지 않음

**외래 키의 주요 제약 조건**
- 참조 무결성 제약
  - 참조하는 테이블의 기본 키와 일치하거나, 값이 NULL 이여야 함
  - 부모 테이블에 없는 값은 넣을 수 없음
 - CASCADE 옵션
  - CASCADE : 부모 데이터 삭제/수정시 자식 데이터도 똑같이 삭제/수정됨
  - SET NULL : 부모 데이터 삭제 시 자식 데이터의 외래 키를 NULL로 번경
  - REESTRICT : 자식 데이터가 있으면 부모 데이터 수정 불가능
  - SET DEFAULT : 부모 데이터 삭제 시, 미리 설정한 기본값으로 번경

## RDBMS 종류
- Oracle Database
- Mysql
