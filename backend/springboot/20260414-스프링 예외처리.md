# 스프링 예외처리

## 1. 에러 코드 정의(열거형)
- `global/error/ErrorCode.java`
```
@AllArgsConstructor
@Getter
public enum ErrorCode {
    DUPLICATE_ID_ERR(401, "DUPLICATE_ID"),

    INTERNAL_SERVER_ERR(500, "INTERNAL_SERVER");

    private Integer errcode;
    private String errormsg;
}
```
위와 같이 열거형으로 에러 코드를 정의함
나중에 코드에서는, `ErrorCode.DUPLICATE_ID_ERR` 와 같이 불러온다. (getter 메서드 사용 가능)

## 2. 에러 응답(DTO)
- **실제로 에러를 클라이언트에 보내주는 DTO**
```
@Builder
public record ErrorResponse(
        Integer errorCode,
        String errorMsg,
        String errorDescription
) {
    public static ErrorResponse errorCodeFrom(ErrorCode errorCode, String description) {
        return new ErrorResponse(
            errorCode.getErrcode(),
            errorCode.getErrormsg(),
            description
        );
    }
}
```
에러 코드를 인자로 받아서, 각 필드에 분해해서 넣어준다.
- **왜 생성자를 쓰지 않고 위와 같이 쓰는가?**
  - 위와 같은 `Of`, `From` 과 같은 함수를 정적 팩토리 메서드 함수라고 한다.
  - 코드의 가독성을 좋게 하기 위해서(생성자를 사용하면 메서드의 목적이 명확하게 드러나지 않음)

## 3. 실제 던지는 에러(비지니스 에러)
```
@Getter
@AllArgsConstructor
public class BusinessException extends RuntimeException {
    private final ErrorCode errorCode;
    private final String description;
}
```
비지니스 에러의 부모 클래스
에러를 처리할 때 모든 에러를 처리하기 어려움
따라서 부모 에러 클래스를 만들어 두고, 상속받는 모든 자식들의 에러를 처리하도록 설계함

## 4. 비지니스 에러를 상속받는 에러
```
public class DuplicateIdErr extends BusinessException {
    public static final BusinessException EXCEPTION = new DuplicateIdErr();
    public DuplicateIdErr() {
        super(ErrorCode.DUPLICATE_ID_ERR, "Id is duplicated");
    }
}
```
비지니스 에러에 직접 넣으면 가독성이 좋지 않아서,
따로 자식 클래스를 만듦
- 또한 `public static final BusinessException EXCEPTION = new DuplicateIdErr();` 과 같이 사용해서, 매번 같은 에러를 `new`로 만들지 않아 메모리 낭비도 적음
- 게시판 없음 에러, 아이디 중복 에러 등을 따로 각각 만들게 됨

## 5. GlobalExceptionHandler
**실제 에러를 처리하는 클래스**
```
@RestControllerAdvice
public class GlobalExceptionHandler {
    @ExceptionHandler(BusinessException.class)
    public ResponseEntity<ErrorResponse> handleBusinessException(BusinessException e) {
        ErrorCode statusCode = e.getErrorCode();
        ErrorResponse response = ErrorResponse.errorCodeFrom(statusCode, e.getDescription());
        return new ResponseEntity<>(response, HttpStatusCode.valueOf(statusCode.getErrcode()));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleException(Exception e) {
        ErrorResponse response = ErrorResponse.errorCodeFrom(ErrorCode.INTERNAL_SERVER_ERR, "Server Error");
        return new ResponseEntity<>(response, HttpStatus.INTERNAL_SERVER_ERROR);
    }
}
```
- `@ExceptionHandler(BusinessException.class)`
  - 아까 만들었던 비지니스 에러를 상속받는 모든 에러들을 처리함
  - `ResponseEntity`로 리턴하는데, 아래 순서대로 인자를 받음.
    1. 실제 에러 DTO
    2. 에러 코드
  - 에러 코드를 직접 지정해서 줄 수 있음
- `@ExceptionHandler(Exception.class)`
  - 따로 정의하지 않은 에러들을 한번에 처리함
