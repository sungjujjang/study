# AOP - Class
만일 JWT와 같이 AOP에서 검증을 완료한 이후 SUB와 같은 값을 어떻게 전달하는가?
- AOP에서 통과 후 컨트롤러에서 다시 검증할 경우 메모리/리소스 낭비가 2배

`HttpServletRequest`**를 사용한다!*

## HttpServletRequest 란?
클라이언트가 서버에 보낸 모든 정보가 담긴 객체
- URL 뒤 쿼리스트링
  - `request.getParameter("id")`
- HTTP 헤더
  - `request.getHeader("Authorization")`
  - `request.getCookies()`
  - 쿠키 정보, 헤더 정보 등을 받아온다.
- 요청 방식(메서드), 주소
  - `request.getMethod()`
  - `request.getRequestURI()`
- 바디 내용(실제로 사용은 DTO딴에서)
  - `request.getInputStream()`

**HttpServletRequest.setAttribute** 를 사용해 전달하게 된다
- 한 요청에서 어디에서 접근하든 값을 공유하는 저장소

## 코드 구조
`AOP`
```
HttpServletRequest request = attributes.getRequest();

String jwtToken = request.getHeader("Authorization");
String userId = jwtSetting.checkToken(jwtToken);
request.setAttribute("userId", userId);
```
AOP에서 검증해 받아온 userId 스트링을 setAttuibute로 등록한다.

`Controller`
```
@GetMapping("/test")
public String testJwt(HttpServletRequest request) {
    String userId = (String) request.getAttribute("userId");
    return userId;
}
```
컨트롤러딴에서 AOP에서 등록한 Attribute를 getAttribute로 받아온다.
