# AOP
관점 지향 프로그래밍의 약어.
스프링에서는 실제 메서드 앞뒤에 공통적인 로직을 구현함
(JWT 인증 로직 등의 공통된 로직)
`@Aspect` - AOP 클래스 맨위에 붙임


## Pointcut
```
@Pointcut("execution(* com.sungjujjang.ter..*Controller.*(..))")
private void cut(){}
```
- Pointcut 지정
  - 예시: 마커를 찍어 AOP가 실행될 위치를 지정함

## Before 어노테이션
```
@Before("cut() && !@annotation(com.sungjujjang.ter.global.NoAuthAnno)")
public void beforeParameterLog(JoinPoint joinPoint) {
	System.out.println("Requested");
	ServletRequestAttributes attributes = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
	if (attributes != null) {
	    HttpServletRequest request = attributes.getRequest();

	    String jwtToken = request.getHeader("Authorization");
	    System.out.println("jwtToken value = " + jwtToken);
	}

	Object[] args = joinPoint.getArgs();
	if (args.length <= 0) System.out.println("no parameter");
	for (Object arg : args) {
	    if (arg != null) {
	        System.out.println("parameter type = " + arg.getClass().getSimpleName());
	        System.out.println("parameter value = " + arg);
	    }
}
}
```
컨트롤러가 실행되기 전에, 미리 실행함
jwt가 유효한지, 유저가 있는지 등 검사함
```
HttpServletRequest request = attributes.getRequest();
String jwtToken = request.getHeader("Authorization");
```
위와 같이 헤더에서 데이터를 받아올 수 있음

## 커스텀 어노테이션
```
@Target({ElementType.METHOD, ElementType.TYPE})
@Retention(RetentionPolicy.RUNTIME)
public @interface NoAuthAnno {
}
```
커스텀 어노테이션이 붙은 메서드(컨트롤러)는 AOP가 실행되지 않게 지정이 가능함
`annotation(com.sungjujjang.ter.global.NoAuthAnno)`
- 원래는 어노테이션이 붙었는지 검사, 앞에 NOT(!)을 붙여서 붙지 않았을때만 실행


## Advice(@Before 등) 실행 시점 어노테이션
- `@Before` 메서드 실행 바로 이전에 로직 실행
- `@After` 메서드 생성 이후에 바로 로직 실행
- `@Around` 메서드 생성 전후 실행 중 등 모두 제어 가능

## Pointcut 표현식 문법
`execution(리턴타입 패키지.클래스.메서드(파라미터))`
- execution이 가장 많이 쓰임
	1. 리턴타입 : `*`(모든 타입), void, string
	2. 패키지.클래스 : `com.example..` 이하 클래스의 모든 패키지, `*Controller` 이름이 컨트롤러로 끝나는 모든 모든 클래스
	3. 메서드(파라미터)
		- `*(..)` 모든 메서드명, 모든 파라미터 종류/수 
		- `*(*)` 파라미터가 정확히 1개인 메서드
		- `get*(..)` get으로 시작하는 모든 메서드
