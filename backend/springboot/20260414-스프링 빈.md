# 스프링 빈
스프링 컨테이너가 관리하는 **자바 객체**이다.
- `new` 키워드 대신 사용함
- 스프링에서 `new` 키워드르 사용시 요청마다 메모리 공간이 낭비됨.
- 싱글톤 패턴으로 미리 new 키워드로 만든 다음 하나의 객체를 돌려 씀

## 빈 등록 방법
- `@Bean` 어노테이션 사용
- `@Component, @Controller, @Service, @Repository` 어노테이션 사용

## 의존 관계란?
**한 클래스가 작업을 수행하기 위해 다른 클래스를 필요로 하는 상태**
```
public class Chef {
    private Knife knife = new Knife(); // Chef가 직접 Knife를 생성 (강한 결합)
}
```
스프링이 아닐 경우, 위와 같이 new 키워드를 통해 만듦.
- 하지만 스프링 컨테이너에서 빈을 가져와갸 하기 때문에 의존 관계 주입(DI)가 필요함

## 생성자 주입으로 의존 관계 구현
```
@Service
@RequiredArgsConstructor // final 필드를 모아서 생성자를 자동 생성
public class MemberService {
    private final MemberRepository memberRepository;
    private final DiscountPolicy discountPolicy;

    // 생성자가 없어도 스프링이 자동으로 주입해줌!
}
```
- `@RequiredArgsConstructor` 을 붙이면 `final`이 붙은 필드롤 모아서 생성자를 만들어 줌
- `final`이 붙은 필드에 의존 관계 주입

## @ComponentScan, @Configuration
1. @Configuration
해달 클래스가 환경 설정 클래스임을 표시
클래스 내부에서 `@Bean` 어노테이션을 사용해 빈을 만들 수 있음
```
@Configuration
public class AppConfig {
    @Bean
    public PasswordSetting passwordSetting() {
        return new PasswordSetting(); // 이 객체는 스프링이 관리하는 'Bean'이 됨
    }
}
```

2. @ComponentScan
특정 패키지를 훑으며 스프링 빈을 찾아내 등록하는 메서드
처음 시작 파일에만 있어 굳이 작성할 필요 없음
