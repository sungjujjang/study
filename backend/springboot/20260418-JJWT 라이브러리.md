# JJWT
Java JWT 라이브러리
자바에서 JWT 토큰을 사용할 수 있도록 하는 라이브러리.
```
implementation 'io.jsonwebtoken:jjwt-api:0.12.3'
runtimeOnly 'io.jsonwebtoken:jjwt-impl:0.12.3'
runtimeOnly 'io.jsonwebtoken:jjwt-jackson:0.12.3'
```
 
## 시크릿 키
```
@Value("${jwt.secret}")
private String secretKeyString;

private SecretKey secretKey;

@PostConstruct
public void init() {
    this.secretKey = Keys.hmacShaKeyFor(secretKeyString.getBytes(StandardCharsets.UTF_8));
}
```
- `@Value("${jwt.secret}")`
  - application.properties 파일에서 jwt.secret 항목을 가져와 넣어준다.

- `@PostConstruct`
  - 빈이 등록될 때 처음 한 번만 실행하게 된다.
  - 모든 필드가 로딩된 후 실행되기 때문에 더 안전함.

- `Keys.hmacShaKeyFor`
  - jjwt 라이브러리에서 시크릿 키를 사용할 수 있도록 변환해준다.


## 토큰 인코딩
```
public String createToken(String userId, Long expirationTime) {
    Date now = new Date();
    Date expDate = new Date(now.getTime() + expirationTime);

    return Jwts.builder()
            .subject(userId)
            .issuedAt(now)
            .expiration(expDate)
            .signWith(secretKey)
            .compact();
}
```
- `.subject`
  - 페이로드의 sub 설정
- `.issuedAt(now)`, `.expiration(expDate)`
  - 토큰이 만들어진 시각(iat), 토큰이 만료될 시각(exp) 를 지정함
- `.signWith(secretKey)`
  - 시크릿 키로 인코딩함.

## 토큰 디코딩
```
public String checkToken(String token) {
    try {
        return Jwts.parser()
                .verifyWith(secretKey)
                .build()
                .parseSignedClaims(token)
                .getPayload()
                .getSubject();
    } catch (JwtException e) {
        throw NotValidJwtErr.EXCEPTION;
    }
}
```
Jwts.parser을 통해서 디코딩한다
```
.getPayload()
.getSubject();
```
위 메서드들로 페이로드 -> sub 순서로 가져오게 된다
최종적으론 sub의 value가 string 형태로 반환된다.

과정에서 **기간이 만료되거나, 검증에 실패(조작된 값)**하면 자동으로 JwtException을 뱉어준다
따라서 밑에서 JwtException을 처리하는 에러 핸들러를 구현했다.
