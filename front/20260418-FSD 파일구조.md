# FSD 구조
아키텍쳐 방법론
리액트에서 파일 구조를 짜는 가장 대표적인 방식

## 계층 구조
**코드를 layer, slices, segments 라는 세 레벨로 구분한다.**
### 층(Layer)
**상단에서 하단으로 흐르는 구조, 위쪽 계층은 아래쪽 계층만 참조할 수 있다**
1. app
  - 애플리케이션 전체 설정(Provider, Router, 전역 스타일 등)
2. processes
  - 여러 페이지에 걸친 복잡한 비지니스 로직
3. pages
  - 애플리케이션의 각 화면
4. widgets
  - 독립적으로 사용 가능한 UI 블록(헤더, 사이드바, 하단바 등등)
5. features
  - 사용자의 가치 있는 행동(로그인, 검색, 리뷰 작성 등)
6. entities
  - 비지니스 실체(User, Product, Order 등 데이터 중심 단위)
7. shared
  - 재사용 가능한 공통 컴포넌트, API 설정 등
```
src/
├── app/                    # 엔트리 포인트, 전역 설정
│   ├── providers/          # Redux, Query, Theme 등의 Provider
│   ├── styles/             # 전역 스타일링
│   └── App.tsx
├── pages/                  # 애플리케이션 페이지 (코드 분할의 단위)
│   ├── home/
│   └── product-detail/
├── widgets/                # 큰 UI 조각 (Page를 구성하는 부품)
│   ├── header/
│   └── product-list/
├── features/               # 비즈니스 가치를 지닌 기능 단위
│   ├── auth-by-phone/      # 전화번호 인증 기능
│   ├── add-to-cart/        # 장바구니 담기 기능
│   └── search-products/    # 상품 검색 기능
├── entities/               # 비즈니스 데이터(모델) 중심
│   ├── user/
│   ├── product/
│   └── order/
└── shared/                 # 특정 도메인에 종속되지 않은 공통 요소
    ├── ui/                 # Button, Input 등 기초 UI 컴포넌트
    ├── api/                # Axios 인스턴스 등
    └── lib/                # 유틸 함수
```

## Slices & Segments
각 계층 내부(특히 entities, features)에서 기능별로 **slice**가 나뉘고,
그 안에는 **segment**로 구성된다.
- ex) `entities/user` 폴더 내부
  - `ui/`, `model/`, `api/` 등등
