# JWT
Json Web Token의 약어
로그인 인증과 같이 세션 서버 없는 인증 로직을 구현할 수 있다.
헤더, 페이로드, 시그니처로 나누어진다.
각 부분 사이사이는 .(점)으로 나눈다.

## 헤더
```
{
  "alg": "HS256"
}
```
이 토큰의 기본적인 정보
- 암호화 방식
- 토큰 유형
등을 담고 있다.

## 페이로드
```
{
  "sub": "wefwevwvewdssdfsfsdf",
  "iat": 1776498998,
  "exp": 99999999999
}
```
생성일자, 만료일자, sub(유저 식별자)를 담는 실제 내용이다.

## 시그니처
앞의 요소들이 번경되지는 않았는지, 검증하는 로직이다
실제로는 시크릿 키가 있어 같이 검증하게 된다.
