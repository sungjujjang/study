# 제네릭

## 제네릭이란
클래스나 메소드에 사용할 내부 데이터 타입을 미리 지정
- 제네릭 없이 타입 다 받으면(Object) 형변환 해야 함
- 제네릭은 형변환이 없어도 됨
```
class MyArray<T> {
    T element;
    void setElement(T element) { this.element = element; }
    T getElement() { return element; }
}
```
위에서 사용한 T를 타입 변수라고 함.
- 임의의 참조형 타입 의미
- 어떤 문자를 사용해도 OK

**위와 같이 선언된 제너릭 클래스를 생성 시, 실제 타입을 명시해야 함**
`MyArray<Integer> myArr = new MyArray<Integer>();`

**모든 제네릭은 컴파일 시에 제거된다.**

## 제네릭 메소드
`public static <T> void sort( ... ) { ... }`
선언부에 타입 변수를 사용한 메소드

## 제네릭 인터페이스
```
interface ExInterfaceGeneric<T> {
    T example();
}

class ExGeneric implements ExInterfaceGeneric<String> {

    @Override
    public String example() {
        return null;
    }
}
```
인터페이스도 제네릭으로 설정 가능

## 제네릭 상속
`T extends Number`
T를 받지만, 일단 Number의 자식들만 받음
- 계산 메서드를 모두 사용가능

## 와일드카드 타입 (`?`)
메서드 내부에서 타입을 다시 부르지 못함
- 그냥 읽기만 할 때는 `?`로
```
// 이름이 없어서 리턴 타입에 쓸 수도 없음 (보통 void나 Object)
public void printList(List<? extends Number> list) {
    for (Number n : list) {
        System.out.println(n);
    }
}
```
**T와의 차이점은 T에서는 타입을 메서드에서 부를 수 있음**
```
public <T extends Number> T getFirst(List<T> list) {
    T item = list.get(0);
    return item;
}
```
