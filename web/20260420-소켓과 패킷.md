# 소켓 (Socket)
두 프로그램이 네트워크를 통해 서로 통신을 수행할 수 있도록 양쪽에 생성되는 창구

**소켓의 종류**
- 스트림(TCP)
  - 양방향으로 바이트 스트림을 전송할 수 있는 **연결 지향형 소켓**
  - 송수신이 가능하다.
  - 오류 수정, 전송 처리, 흐름 제어 등 보장
  - 대량의 데이터를 보내는 경우에 적합
  - 신뢰성이 높다
- 데이터그램(UDP)
  - 명시적으로 연결을 맺지 않은 **비 연결형 소켓**
  - 데이터의 크기에 제한이 있다
  - 데이터를 빠르게 전송 시에 적합
  - 확실하게 전달이 보장되지 않고, 데이터 손실에도 오류가 발생하지 않음

- 서버
  1. 소켓 생성
  2. 아이피와 포트번호 설정
  3. 수신 대기열 생성
  4. 클라이언드 연결 대기

- 클라이언트
  1. 소켓 열기
  2. 지정된 아이피와 포트에 연결
  3. 서버와 통신

# 패킷 (Packet)
네트워크에서 데이터를 주고받을 때 사용되는 데이터 조각
패킷의 3요소
- 헤더
  - 패킷에 대한 메타정보
  - 패킷의 길이, 번호, 프로토콜, 목적지 IP주소
- 페이로드
  - 수신자가 요청한 데이터
- 트레일러
  - 패킷의 끝을 나타내는 항목
  - 오류 검출 코드가 포함되면 데이터 손실 오류를 방지할 수 있다

![패킷의 구조](https://static.tosspayments.com/docs/glossary/packet-example.png)
## 왜 데이터를 통째로 보내지 않는가?
- 전송 과정에서 한번의 오류로 전체 데이터가 손실될 수 있다
  - 손실시 전체 데이터를 다시 전송해 시간이 오래 걸림
- 대용량 데이터를 전송하면 네트워크 혼잡도가 올라가고, 모든 사용자의 속도가 저하
- 작은 데이터로 나누면 서버 용량을 분산할 수 있다
