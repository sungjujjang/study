# WEB
- 정적인 자원을 찾아 바로 응답
- 동적인 자원 처리 불가능, 모든 자원 리스트에서 찾기 때문에 시간이 오래 걸림
- WEB의 단점을 보완하기 위해 WAS가 등장

## WAS
웹 서버와 웹 컨테이너의 결합
- 동적인 자료를 처리하기 위함
- DB 수정, 비즈니스 로직 처리 등
- 실시간으로 페이지에 필요한 파일 생성
**보통 WEB - WAS로 연결하여 사용하기도 함**

## Tomcat
자바 서블릿(Java Servlet), JSP를 실행할 수 있는 환경
- Connector
  - 외부로부터 들어오는 요청(HTTP)을 받아 컨테이너로 전달
- Container
  - 서블릿의 생명 주기 관리
  - 실제 비즈니스 로직(Java 코드)을 실행
- Jasper
  - JSP 파일을 서블릿(Java 코드)으로 변환하고 컴파일하는 엔진
**서블릿은 자바를 사용하여 웹 페이지를 동적으로 생성하는 자바 구조(생성법)**
