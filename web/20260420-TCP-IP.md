# TCP/IP
인터넷에서 컴퓨터들이 서로 데이터를 주고받을 수 있는 프로토콜의 집합

## TCP
신뢰성 있는 데이터 전송 방식
**3-Way Handshake 란?**
- TCP에서 연결을 시작하는 방법
1. A 클라이언트는 B 서버에 접속을 요청하는 패킷 전송
2. B 서버는 A 클라이언트에게 요청을 수락한다는 패킷 전송
3. A 클라이언트는 요청 수락을 수락하는 패킷을 전송하고 이후 데이터 통신이 이루어진다.

**4-way Handshaking**
- TCP에서 연결을 종료하는 방법
1. A 클라이언트는 연결 종료 패킷을 전송
2. B 서버는 확인 메시지를 보내고 통신이 끝날 떼 까지 기다림
3. 서버가 통신이 끝났으면 연결이 종료되었다고 클라이언트에게 패킷(FLAG)을 전송한다
4. 클라이언트는 확인 패킷을 보낸다

## IP
컴퓨터에 부여된 고유 주소를 바탕으로 목적지까지 데이터가 찾아갈 수 있는 길

**TCP/IP**는 4개의 계층으로 나뉘어 작동한다.
데이터가 상위 계층에서 하위 계층으로 내려가며 캡슐화되고,
받을 때는 아래에서 위로 점점 역캡슐화한다.
4. 응용 계층
  - 사용자가 사용하는 소프트웨어와 작접 통신
  - HTTP, FTP, SMTP 등
3. 전송 계층
  - 데이터 전송 방식 결정 및 오류 제어
  - **TCP**, UDP 등
2. 인터넷 계층
  - 패킷을 목적지 주소까지 전달(라우팅)
  - **IP**, ICMP
1. 네트워크 인터페이스
  - 물리적인 네트워크(하드웨어)를 통한 전송
  - 이더넷, 와이파이

## TCP/IP를 사용하는 이유
- 범용성
  - 하드웨어, 운영체제, 네트워크 등에 상관없이 모든 곳에서 사용 가능
- 신뢰성
  - TCP의 확인 절차 덕분에 데이터가 유실될 걱정을 하지 않아도 된다
